# Проект: Автономный AI-исследователь (Deep Research Service)

Ты — Python Developer. Твоя задача — реализовать локальный веб-сервис для автономного поиска и синтеза информации.

## Технологический стек
- **Язык:** Python 3.12, AsyncIO.
- **Фреймворк:** FastAPI (REST API + lifespan events).
- **БД:** Redis (через `redis.asyncio`).
- **Сеть:** `aiohttp` для запросов к SearXNG и Jina Reader API.
- **LLM:** Единый абстрактный слой `app.core.llm` поверх `google-genai` (Gemma 3) и `openai` API (совместимые API).

## Строгие архитектурные правила
1. **Конфигурация:** Только через переменные окружения. Никаких хардкод-значений для URL, портов, таймаутов или ключей.
2. **Изоляция состояний:** Никаких глобальных переменных состояния в памяти. Каждый API-запрос должен работать в своем изолированном контексте.
3. **Отказоустойчивость (Graceful Degradation):** Ни один модуль не должен обрушать весь пайплайн. Ошибки логируются (warning/error), система продолжает работу с доступными данными.
4. **Атомарность Redis:** Для изменения репутации нод использовать строго атомарные операции (`ZINCRBY`), без паттерна read-modify-write.
5. **Интеграция с LLM:** При запросе JSON всегда использовать `response_mime_type="application/json"` (для Gemini) или аналоги. Обязательно реализовывать защиту от галлюцинаций.
6. **Абстрактный слой с LLM:** Все модули (Задачи 4, 5, 7) должны использовать `app.core.llm` для генерации текста. Прямой импорт SDK провайдеров (`google.genai`, `openai`) в бизнес-логике строго запрещен!

Перед выполнением любой задачи сверяйся с соответствующим файлом ТЗ в папке `docs/`.
