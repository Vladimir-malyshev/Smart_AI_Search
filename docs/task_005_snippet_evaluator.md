# Задача 5: AI-Триаж и фильтрация сниппетов (Snippet Evaluator)

## Контекст и цель

Экономия токенов и времени парсинга. Модуль принимает «грязную» поисковую выдачу (20–40 сниппетов) и через LLM выбирает только те URL, которые с наибольшей вероятностью содержат ответ на цель пользователя.

---

## Конфигурация

```
GEMINI_API_KEY=...
GEMINI_MODEL=gemini-2.5-flash
EVALUATOR_MAX_SELECTED=4          # максимум URL на выходе
EVALUATOR_MIN_SELECTED=1          # минимум (если 0 — значит ничего не найдено, это ок)
```

---

## Входные / выходные данные

- **Вход:** строка цели (Goal) + массив из 20–40 объектов `{url, title, snippet}`
- **Выход:** строгий JSON: `{"selected_urls": ["url1", "url2"]}`

---

## Основная логика

### Шаг 1: Подготовка данных

Преобразовать массив сниппетов в компактный нумерованный список для передачи в LLM:

```python
def format_snippets_for_llm(snippets: list[SearchSnippet]) -> str:
    lines = []
    for i, s in enumerate(snippets):
        lines.append(f"[{i}] {s.title}\nURL: {s.url}\n{s.snippet}\n")
    return "\n".join(lines)
```

### Шаг 2: Системный промпт

```
Ты — информационный аналитик. Тебе дан список поисковых сниппетов и цель пользователя.

Твоя задача: выбрать максимум {EVALUATOR_MAX_SELECTED} URL, которые с наибольшей 
вероятностью содержат информацию для достижения цели.

Правила отбора:
- Отдавай предпочтение первичным источникам: официальная документация, исследования, 
  новостные статьи от изданий, блоги экспертов
- Строго игнорировать: Pinterest, агрегаторы без контента, кликбейт-заголовки, 
  SEO-дорвеи ("скачать бесплатно", "топ 100 сайтов"), Reddit-треды без ответов

Ответ строго в формате JSON, без пояснений:
{"selected_urls": ["https://...", "https://..."]}

Если ни один сниппет не релевантен — вернуть: {"selected_urls": []}
```

### Шаг 3: Вызов API и парсинг

```python
from app.core.llm import llm_provider

async def evaluate_snippets(goal: str, snippets: list[SearchSnippet]) -> list[str]:
    formatted = format_snippets_for_llm(snippets)
    prompt = f"Цель: {goal}\n\nСниппеты:\n{formatted}"
    
    # Использование абстрактного провайдера
    response_text = await llm_provider.generate_json(
        prompt=prompt,
        system_prompt=SYSTEM_PROMPT
    )
    
    raw_urls = parse_selected_urls(response_text)
    
    # Анти-галлюцинация: оставить только URL, реально присутствующие во входном массиве
    valid_input_urls = {s.url for s in snippets}
    verified = [url for url in raw_urls if url in valid_input_urls]
    
    hallucinated = set(raw_urls) - set(verified)
    if hallucinated:
        logger.warning(f"LLM выдумала {len(hallucinated)} URL, они отброшены: {hallucinated}")
    
    return verified

## Обработка исключений

### LLM выдумала URL (галлюцинация)

После парсинга ответа каждый URL сверяется с исходным массивом сниппетов. URL, которого нет во входных данных, молча отбрасывается. Это обязательная проверка, не опциональная.

### Выбрано 0 URL

Допустимый результат — означает, что LLM не нашла ничего релевантного. Оркестратор (Задача 8) обработает этот случай и может инициировать новый поисковый цикл.

## План тестирования (Критерии приёмки)

### Тест 1: Анти-галлюцинация

```
Дано: 5 сниппетов с реальными URL (url_1 ... url_5)
Ситуация: LLM в ответе возвращает url_1, url_3 и url_invented (несуществующий)

Ожидаемый результат:
  - url_1 и url_3 присутствуют в итоговом списке
  - url_invented отброшен
  - в логах: WARNING о галлюцинации
```

### Тест 2: Фильтрация мусора

```
Дано: массив из 10 сниппетов:
  - 1 сниппет: официальная документация LangChain (релевантна цели)
  - 9 сниппетов: SEO-мусор ("скачать бесплатно", "топ 50 библиотек без регистрации")
  
Цель: "Как использовать LangChain для построения RAG-пайплайна"

Ожидаемый результат:
  - selected_urls содержит URL документации LangChain
  - SEO-мусор не выбран
```
