Задача: Внедрение финального LLM-Синтезатора в E2E тесты

Нам необходимо обновить скрипт автоматического тестирования (предположительно run_automated_tests.py или файл, где запускаются тесты пайплайна). Цель — симулировать поведение клиентской LLM, которая получает данные от нашего RAG-API и отвечает пользователю.

Пожалуйста, внеси следующие изменения в скрипт тестов:

1. Новая функция synthesize_final_answer
Создай асинхронную функцию, которая принимает user_query (оригинальный запрос), user_goal (цель) и scraped_markdown (полный текст собранных статей).
Для генерации ответа используй провайдера gemini и модель gemini-3.1-flash-lite-preview (или актуальную версию Flash).

2. Строгий System Prompt (Жесткая привязка к контексту)
Используй следующий системный промпт для Flash-модели:
"""
Ты — строгий QA-ассистент. Твоя единственная база знаний — это текст, предоставленный ниже в блоке [МАТЕРИАЛЫ]. Ты не знаешь ничего о реальном мире.
Твоя задача — дать подробный и развернутый ответ на вопрос пользователя, чтобы достичь его цели.

ПРАВИЛА:

    Используй ТОЛЬКО факты из предоставленного текста.

    Если в тексте есть ответ — напиши понятный, развернутый ответ.

    Если в тексте НЕТ нужной информации для ответа, ты ОБЯЗАН ответить: "ОТКАЗ: В собранных материалах нет ответа на запрос".

    ЗАПРЕЩЕНО додумывать, использовать внутреннюю память или писать "в тексте нет, но на самом деле...".
    """

3. Формирование User Prompt
Склей запрос так:
f"Запрос: {user_query}\nЦель: {user_goal}\n\n[МАТЕРИАЛЫ]:\n{scraped_markdown}"

4. Интеграция в цикл тестирования
В конце каждого теста, когда пайплайн возвращает status: complete (или завершает все итерации), склеивай содержимое всех успешных URL в одну Markdown-строку scraped_markdown и передавай в synthesize_final_answer.
Выводи результат синтезатора в лог и сохраняй в итоговый отчет/файл теста (например, в тот же .md файл, где лежат сырые данные, добавь блок # Финальный ответ LLM).
