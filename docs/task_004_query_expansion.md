# Задача 4: AI-Планировщик (Query Expansion & Goal Analysis)

## Контекст и цель

Первое обращение к LLM в пайплайне. Модуль переводит «сырое» намерение пользователя в набор высокоэффективных машинных поисковых запросов, пригодных для отправки в SearXNG.

---

## Конфигурация

```
GEMINI_API_KEY=...
GEMINI_MODEL=gemma-3-27b-it          # старшая модель линейки Gemma 3; уточнить актуальный идентификатор в документации Google
QUERY_MAX_WORDS=10                      # максимум слов в одном поисковом запросе
QUERY_MIN_COUNT=2                       # минимум запросов в ответе
QUERY_MAX_COUNT=5                       # максимум запросов в ответе
```

**Примечание по модели:** использовать старшую модель линейки Gemma 3 через Google Gemini API. Актуальный идентификатор модели уточнить в документации Google на момент разработки — брать из переменной `GEMINI_MODEL`, не хардкодить.

---

## Входные / выходные данные

- **Вход:** строка запроса пользователя + строка цели (Goal)
- **Выход:** строгий JSON: `{"queries": ["q1", "q2", "q3"]}`

---

## Основная логика

### Системный промпт

```
Ты — старший исследователь данных. Твоя задача — превратить намерение пользователя 
в набор поисковых запросов для поисковой системы.

Правила генерации запросов:
1. Один запрос должен быть широким (общий контекст темы)
2. Один запрос должен быть узкоспециализированным (профессиональные термины, цифры, даты)
3. Если тема касается технологий, бизнеса или науки — один запрос обязательно на английском 
   языке для охвата западных источников
4. Каждый запрос — не более {QUERY_MAX_WORDS} слов
5. Никаких пояснений, только JSON

Ответ строго в формате:
{"queries": ["запрос1", "запрос2", "запрос3"]}
```

### Вызов API

Вместо прямого импорта SDK, использовать абстракцию из Задачи 3.1.

```python
from app.core.llm import llm_provider

async def expand_query(user_query: str, goal: str) -> list[str]:
    user_message = f"Запрос пользователя: {user_query}\nЦель: {goal}"
    
    # Вызов абстрактного фабричного метода, который гарантирует JSON
    response_text = await llm_provider.generate_json(
        prompt=user_message,
        system_prompt=SYSTEM_PROMPT

### Валидация ответа

```python
def validate_and_parse(raw: str) -> list[str]:
    # Защита от "Вот ваши запросы: ```json {...}```"
    clean = re.sub(r"```json|```", "", raw).strip()
    
    data = json.loads(clean)  # выбросит исключение если невалидный JSON
    queries = data.get("queries", [])
    
    if not isinstance(queries, list):
        raise ValueError("queries должен быть массивом")
    
    if not (QUERY_MIN_COUNT <= len(queries) <= QUERY_MAX_COUNT):
        raise ValueError(f"Ожидается от {QUERY_MIN_COUNT} до {QUERY_MAX_COUNT} запросов, получено {len(queries)}")
    
    # Обрезка слишком длинных запросов
    trimmed = []
    for q in queries:
        words = str(q).split()
        trimmed.append(" ".join(words[:QUERY_MAX_WORDS]))
    
    return trimmed
```

## Обработка исключений

### Битый JSON в ответе LLM

Если `json.loads()` бросает исключение:
- Попробовать извлечь JSON через `re.search(r'\{.*\}', raw, re.DOTALL)`
- Если и это не помогло — залогировать ответ модели и выбросить исключение вверх (оркестратор обработает)

### Запросы слишком длинные

Обрезка до `QUERY_MAX_WORDS` слов выполняется всегда, независимо от того, соблюдала ли модель правило.

## План тестирования (Критерии приёмки)

### Тест 1: Семантическое качество запросов

```
Вход:
  query = "Узнать цены на IT разработку в РФ в 2026 году"
  goal = "Сформировать бюджет на создание AI-продукта"

Проверка (ручная или через LLM-as-judge):
  - запросы НЕ являются простым повторением исходной фразы
  - присутствует хотя бы один специализированный запрос (термины, цифры)
  - присутствует хотя бы один запрос на английском языке
  
Примеры ожидаемых запросов:
  "стоимость создания AI продукта Россия 2026"
  "аналитика цен разработка сайтов рунет"
  "rates python developers russia 2026"
```

### Тест 2: Стабильность формата

```
Действие: запустить вызов API 20 раз подряд с одним и тем же входом
(100 итераций нецелесообразны по стоимости, достаточно 20)

Ожидаемый результат:
  - все 20 ответов успешно парсятся через json.loads()
  - процент ошибок парсинга: 0%
```
